üîß How It Works

1. Call Initiation

When a call is received or initiated, Twilio connects to the /voice endpoint of the server.

2. Conversation Flow

The AI system guides the caller through a series of questions:

  1. Name Collection: "Hello! Please say your full name after the beep."
  2. Email Collection: "Now please tell your email address clearly."
  3. Age Collection: "Great! Now please tell me your age clearly."
  4. Location Collection: "Perfect! Now please say your city and state or country clearly."
  5. Introduction: "Finally, give a short introduction about yourself."

3. Data Storage

All collected information is stored in logs.json with timestamps and caller details.

4. Dashboard Display

The web dashboard fetches data from /logs endpoint and displays it in an organized table format with statistics.

üì° API Endpoints

GET /

Serves the main dashboard interface.

POST /voice

Handles incoming voice calls and processes speech input using TwiML responses.

GET /logs

Returns all call logs in JSON format with statistics.

POST /call-me

Initiates a test call to a configured phone number.

GET /test

Health check endpoint to verify server status.

üîê Environment Variables

The application requires the following environment variables:

  • TWILIO_ACCOUNT_SID - Your Twilio account SID
  • TWILIO_AUTH_TOKEN - Your Twilio authentication token
  • TWILIO_PHONE_NUMBER - Your Twilio phone number (from)
  • TEST_PHONE_NUMBER - Phone number for test calls (to)
  • BASE_URL - Your server's public URL (for webhooks)
  • PORT - Server port (default: 5500)
